## What ArchBoard Does

ArchBoard pulls together the everyday configuration work for Arch Linux and Hyprland into one web-driven workspace. Rather than jumping between multiple files and shell commands, you can inspect and modify system settings from a single interface that is meant to be fast to navigate and easier to keep organized.

It is a strong fit for anyone looking for a more convenient way to manage desktop configuration and system tuning. With real-time configuration updates and built-in backup creation, ArchBoard reduces repetitive manual steps while making changes simpler to track and handle.

## Using ArchBoard

ArchBoard is intended for checking Hyprland settings, changing system configuration values, and saving backups before making broader edits. A common workflow looks like this:

1. Open the ArchBoard web interface
2. Review the current configuration state
3. Edit the relevant system or Hyprland settings
4. Apply changes and wait for real-time updates
5. Save a backup of the active configuration when needed

For routine maintenance, come back to the interface whenever you want to refine desktop behavior or recover an earlier setup.

## Configuration Layout

ArchBoard is built around configuration management, so settings are controlled through the web interface and the underlying Arch Linux installation.

If you store local settings in files, a simple structure such as the following works well:

config/
  hyprland/
  system/
  backups/

Use backups before applying major changes, especially when adjusting desktop behavior or system-level values.

## FAQ

**How do I get updates?**  
Use the latest build link above or check the repository for new releases.

**Can I keep a copy of my current setup?**  
Yes. ArchBoard includes backup support for current configurations.

**Where are settings changed?**  
Settings are managed through the web interface and applied to Hyprland or system configuration targets.

**What should I do if changes do not appear right away?**  
Refresh the interface, confirm the target setting was applied, and verify the configuration source used by your Arch Linux setup.

**Is this meant only for Hyprland?**  
Hyprland is a primary focus, but ArchBoard also covers essential system settings.
